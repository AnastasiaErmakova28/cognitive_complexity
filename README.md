## О проекте

Репозиторий содержит программную реализацию научно-исследовательского проекта по методам оценки и алгоритмическому снижению **когнитивной сложности** русскоязычных текстовых материалов.

Исследование решает проблему когнитивной перегрузки при чтении специализированной документации. В отличие от традиционных методов **лексического упрощения** (замены сложных слов на простые), которые ведут к потере смысла и фактологическим ошибкам, предложенный подход фокусируется на **синтаксических изменениях** — перестройке структуры предложений без искажения технического содержания.

## Ключевая идея

Алгоритм снижает два ключевых показателя синтаксической сложности:

- **MDD** (Mean Dependency Distance) — среднюю длину синтаксических зависимостей;
- **уровень полипредикативности** — количество предикативных центров в предложении.

Это уменьшает нагрузку на рабочую память читателя, **сохраняя исходный технический смысл**.

Для оценки эффекта применяется набор классических и адаптированных под русский язык метрик читабельности (см. раздел [Метрики](#метрики-читабельности)).

## Быстрый старт

Запустить Jupyter и открыть нужный ноутбук:

```bash
jupyter notebook Metrics.ipynb
```

Минимальный пример расчёта одной из метрик:

```python
import re, math

def count_syllables_ru(word):
    vowels = "аеёиоуыэюя"
    return max(1, sum(1 for ch in word.lower() if ch in vowels))

def fres_ru(text):
    sents = [s for s in re.split(r'[.!?]+', text) if s.strip()]
    words = re.findall(r'\b\w+\b', text.lower())
    if not words or not sents:
        return 0.0
    wps = len(words) / len(sents)
    spw = sum(count_syllables_ru(w) for w in words) / len(words)
    return 206.835 - 1.3 * wps - 60.1 * spw

print(fres_ru("Некоторый технический текст на русском языке."))
```

## Метрики читабельности

В [Metrics.ipynb](Metrics.ipynb) реализованы следующие формулы, адаптированные под русский язык:

| Метрика | Функция | Что измеряет |
|---|---|---|
| **FRES** — Flesch Reading Ease | `fres_ru(text)` | Лёгкость чтения (чем выше, тем проще) |
| **FKGL** — Flesch-Kincaid Grade Level | `fkgl_ru(text)` | Соответствие классу обучения |
| **SMOG** | `smog_ru(text)` | Индекс «тумана» по многосложным словам |
| **ARI** — Automated Readability Index | `ariru(text)` | Индекс по символам и предложениям |
| **CLI** — Coleman-Liau Index | `cliru(text)` | Индекс на основе длины слов |
| **TI** | `ti_ru(text)` | Индекс средней длины слов и предложений |
| **LIX** | `lix_ru(text)` | Индекс шведской читабельности |
| **MDD** | см. финальный ноутбук | Средняя длина синтаксических зависимостей |
| **Полипредикативность** | см. финальный ноутбук | Число предикативных центров |

Все функции принимают `str` и возвращают `float`.

## Данные

Для валидации алгоритма использовалась **краудсорсинговая оценка читателей** через Яндекс.Толоку. В репозитории лежат агрегированные результаты двух пулов:

- `aggregated_results_by_ds__pool_37806285__2023_02_18.tsv`
- `aggregated_results_by_ds__pool_37939772__2023_02_27.tsv`

Формат — TSV, по колонкам: идентификатор текста, оценки сложности, метаданные пула.

## Результаты

Предложенный подход снижает показатели MDD и полипредикативности **без потери технического содержания**, что подтверждается как автоматическими метриками читабельности, так и пользовательскими оценками (см. TSV-файлы и [domain_analysis.ipynb](domain_analysis.ipynb)).
